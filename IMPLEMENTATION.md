# Что реализовано

Файл фиксирует текущее состояние backend-проекта «Simple Online Store» (Node.js + Express + PostgreSQL + Sequelize). Он поможет вернуться к точке, на которой остановились, и понять, какие компоненты уже подняты.

## Общее устройство
- **Точка входа:** `index.js` заводит Express по порту `process.env.PORT || 5000`, подключает CORS, парсит JSON, обрабатывает файлы через `express-fileupload` и раздаёт папку `static`. После успешной аутентификации Sequelize (`sequelize.authenticate()` + `sequelize.sync({ alter: true })`) запускается маршрутизатор `/api`.
- **Конфигурация БД:** `server/db.js` читает `.env`, экспортирует Sequelize с параметрами из окружения (`DB_NAME`, `DB_USER`, `DB_PASS`, `DB_HOST`, `DB_PORT`) и `underscored: true` для snake_case.
- **Модульная архитектура:** `server/` разбит на `routes`, `controllers`, `models`, `services`, `middleware` и `errors`. `server/models/mapping.js` создаёт таблицы `Product`, `Category`, `Brand`, `User`, `Basket`, `BasketProduct` и связывает их (категории/бренды → продукты, корзина ↔️ товары, пользователь ↔️ корзина).

## Реализованное API
| Контроллер | Методы | Что делает |
| :---------- | :------ | :----------- |
| `/api/product` (полный контроллер) | `GET /getall`, `GET /getone/:id`, `POST /create` | Поддерживает фильтрацию по `brandId`/`categoryId`, пагинацию, загрузку картинки (`req.files.image` сохраняется в `static`), возвращает список, единичный товар и добавление нового товара через `ProductModel`.
| `/api/category` | `GET /getall`, `POST /create` | Простые операции чтения/создания категорий с логированием ошибок.
| `/api/brand` | `GET /getall`, `POST /create` | Аналогично категориям, хранится список брендов.
| `/api/user` | заглушки (`signup`, `login`, `check`, CRUD) | Все ответы пока «placeholder», настоящая авторизация/пользователи не реализованы.

`test.http` содержит запросы (создание категории, бренда, товара) против `http://localhost:7000/api` для ручного тестирования.

## Дополнительно
- **Сервисы и утилиты:** `server/services/FileService.js` умеет сохранять/удалять файлы, но контроллер товара пока напрямую вызывает `image.mv(...)`; есть лишняя пара моделей с `Logger`, которые контроллеры не используют.
- **Middleware:** `Logger` выводит информацию/ошибки с временными метками, `ErrorHandler` (пока не подключён) умеет перехватывать кастомные `AppError`, ошибки Sequelize, JSON/SyntaxError и ограничения по файлам; `authMiddleware`/`adminMiddleware` реализуют JWT и проверку ролей, но тоже не подключены.
- **Модели:** `Brand`, `Category`, `Product` (классы с CRUD и validation) существуют рядом с `mapping.js`, но контроллеры работают напрямую со Sequelize-моделями, поэтому логика дублируется.

## Что ещё не сделано
1. `/api/user` — заглушки вместо реальных контроллеров (`signup`, `login`, `check`).
2. Обновление/удаление товаров/категорий/брендов не подключены (`PUT /update/:id`, `DELETE` возвращают только `200 OK`).
3. `ErrorHandler` и middleware авторизации не зарегистрированы в Express (`app.use(errorHandler)` отсутствует). С переносом JWT это нужно сделать.
4. Папка `static` не в репозитории, но используется для хранения изображений — нужно создать, задокументировать и убедиться, что в prod она доступна.
5. Разноплановая работа с файлами: часть логики лежит в `FileService`, а часть (в контроллере) не используется.
6. Нет `.env.example`, документации по ENV, тестов и миграций — всё ещё не работает как полноценное API.

## Следующие шаги
1. Зарегистрировать `ErrorHandler`, `authMiddleware`, `adminMiddleware`, подключить JWT и реализовать реальные контроллеры `User`.
2. Привести загрузку изображений к `FileService`, добавив удаление при обновлении/удалении товаров.
3. Расширить роуты продукт/категория/бренд реальными `PUT`/`DELETE`, добавить проверки авторизаций.
4. Добавить документацию по ENV, команды запуска (`npm run dev`) и инструкции по тестам (например `test.http`).
